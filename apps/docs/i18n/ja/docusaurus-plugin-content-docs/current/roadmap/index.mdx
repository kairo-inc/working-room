# ロードマップ

このロードマップは WorkingRoom の現在の開発方針と今後の拡張予定エリアを概説します。優先度の変化に応じて内容は変更される場合があります。

## 現在の取り組み

以下の機能が現在進行中です：

| 機能                        | カテゴリ            | 説明                                                                              |
| --------------------------- | ------------------- | --------------------------------------------------------------------------------- |
| アクセスロール管理 UI       | Web UI              | アクセス制御のロール、ポリシー、ユーザー割り当てを作成・管理するインターフェース  |
| 削除ファイルの復元           | ファイル管理        | 誤って削除したファイルをゴミ箱から復元する                                        |

## 予定

以下の項目はエリア別にグループ化され、今後のリリースで予定されています。

### Web UI

- **アクセスロール管理 UI** — アクセス制御のロール、ポリシー、ユーザー割り当てを作成・管理するインターフェース
- **クリップボードからのファイルアップロード** — クリップボードから画像やファイルをワークスペースに直接貼り付ける
- **ブラウザ内テキストエディタ** — WorkingRoom 内でマークダウンやプレーンテキストファイルを直接編集する

### AI とエージェント体験

- **カスタム AI エージェント設定** — 独自の AI エージェントを登録・設定する UI
- **Gemini サポート** — 既存のプロバイダーに加えて Google Gemini モデルとの統合

### ファイルシステム

- **削除ファイルの復元** — 誤って削除したファイルをゴミ箱から復元する
- **拡張ファイル形式サポート** — 追加のファイルタイプ（スプレッドシート、プレゼンテーションなど）のプレビューと操作
- **フォルダのアップロード** — フォルダ構造全体をファイルツリーにドラッグ＆ドロップする
- **ファイル検索** — ファイルシステム全体での全文検索とファイル名検索

### インテグレーション

- **AWS S3 ストレージバックエンド** — デフォルトのストレージではなく、独自の S3 バケットにファイルを保存する

## 将来のアイデア

即時のロードマップには含まれていない長期的なアイデア：

- **マルチテナントデプロイ** — 複数の独立した組織にサービスを提供する共有プラットフォームとして WorkingRoom を運用する
- **Google 認証** — メール/パスワードに加えて Google アカウントでサインインする
