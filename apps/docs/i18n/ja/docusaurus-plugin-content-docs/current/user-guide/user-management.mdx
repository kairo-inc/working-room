# ユーザー管理

WorkingRoom では、テナントにユーザーを招待できます。
同じテナントのユーザーは同じファイルシステムを共有し、同じワークスペースで協働できます。

また、各ユーザーにロールを割り当てて、テナント内で許可される操作を制御できます。

## テナントにユーザーを招待する

テナントが作成されたら、追加のユーザーを招待できます。

ユーザー一覧画面を開いて、テナントの現在のユーザーを確認し、ユーザー関連の操作を管理できます。

ユーザーの削除、ロールの変更、パスワードのリセットなどのアクションメニューを開くには、ユーザー一覧の行を右クリックします。

<Screenshot alt="WorkingRoom ユーザー一覧" light="/img/screenshots/user_list_light.png" dark="/img/screenshots/user_list_dark.png" />

招待されたユーザーは同じテナントのメンバーになります。つまり、そのテナントの共有ファイルシステムにアクセスし、他のユーザーと協働できるようになります。

割り当てるロールによって、ユーザー管理や組織設定へのアクセスを制限できます。

新しいユーザーを招待すると、WorkingRoom は招待モーダルを表示します。そこでユーザー情報を入力し、適切なロールを割り当てられます。

## ロール

WorkingRoom では以下のテナントロールを提供しています：

| 権限                   | `owner` | `admin` | `member` |
| ---------------------- | :-----: | :-----: | :------: |
| ユーザーの招待         |   ✅    |   ✅    |    🚫    |
| ユーザーの削除         |   ✅    |   ✅    |    🚫    |
| ロールの変更           |   ✅    |   ✅    |    🚫    |
| パスワードのリセット   |   ✅    |   ✅    |    🚫    |
| 組織設定を開く         |   ✅    |   ✅    |    🚫    |
| テナント設定の編集     |   ✅    |   🚫    |    🚫    |
| テナントの削除         |   ✅    |   🚫    |    🚫    |

## ロールの詳細

### Owner

`owner` はテナントを作成したユーザーです。

各テナントに `owner` は 1 人のみです。
オーナーはユーザーの管理、組織設定の確認、テナント設定の更新、およびオーナー専用のテナント操作を実行できます。

### Admin

`admin` ロールはテナント管理者向けです。

Admin はユーザーの招待、削除、パスワードのリセット、ロールの変更、組織設定ページの表示など、主要なユーザー管理タスクを実行できます。

実際には、Admin はテナント運用に必要なほぼすべての操作ができます。
ただし、`admin` は `owner` と完全に同一ではなく、一部のテナントレベルの操作はオーナー専用として残ります。

### Member

`member` ロールはテナント内で作業する一般ユーザー向けです。

Member はテナントの共有ファイルシステムを使用できますが、他のユーザーを管理することはできません。
ユーザーの追加・削除、ロールの変更、組織設定ページの表示はできません。

:::note

デフォルトでは、1 つの WorkingRoom サーバーで作成できるテナントは 1 つのみです。
複数のユーザーがそのテナントで作業したい場合は、ユーザーを招待して適切なロールを割り当ててください。

:::
